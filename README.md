## 📌 What is this Project?

This project is a lossless file compressor. Although historically structured under the namespace `huffman` due to initial project requirements, the core engine implements **Lempel-Ziv-Welch (LZW)** compression. It showcases:
1. A native, zero-dependency C++ CLI compiler/decompiler.
2. A premium modern single-page HTML web interface that runs the exact same LZW algorithm in Javascript.
3. Complete **binary compatibility**—files compressed in C++ can be decompressed in the web browser, and vice versa.

## 🏗️ System Architecture & Binary Specification

Both the C++ command-line tool and the JavaScript web visualizer follow a strict binary layout to remain fully compatible.

### Binary Layout (.huff File Format)
```
┌───────────────┬──────────────────┬─────────────────────────────────┐
│  Magic Bytes  │  Version Flag    │       Compressed Code Stream     │
│   'L' 'Z' 'W' │      0x01        │  16-bit Little-Endian Integers  │
│    (3 bytes)  │    (1 byte)      │           (Variable)            │
└───────────────┴──────────────────┴─────────────────────────────────┘
```

### Algorithmic Parameters
* **Initial Dictionary:** 0–255 mapped to ASCII/raw bytes.
* **Special Codes:** Code `256` is reserved as the `RESET_CODE`.
* **Dictionary Expansion Limit:** Up to `65,536` entries (16-bit address space).
* **Reset Strategy:** Once the dictionary size reaches the 16-bit capacity (`65,536`), a `RESET_CODE` is emitted into the stream, and the dictionary is cleared to adapt to new character distribution patterns.

## 🛠️ Tech Stack

* **Core Backend CLI:** C++17 (Standard Library, zero external dependencies).

* **Build System:** CMake (CMakeLists.txt provided for cross-platform builds).

## 🚀 How to Run

### 1. Running the C++ Command-Line Tool

**Build with CMake:**
```bash
mkdir build && cd build
cmake ..
cmake --build . --config Release
```

**Compress a file:**
```bash
huffman_compressor.exe compress input.txt output.huff
```

**Decompress a file:**
```bash
huffman_compressor.exe decompress output.huff restored.txt
```

### 2. Running the Web Application
Simply open `index.html` in any web browser. You can drag and drop any file to compress it locally in the browser, watch the compression ratio stats, inspect dictionary entries, or decompress `.huff` archives generated by the C++ engine.
